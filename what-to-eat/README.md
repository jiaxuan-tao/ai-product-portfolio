# What to Eat｜今天吃什么 🍜

“今天吃什么”是一个帮你结束点餐纠结的轻量 Web 转盘。

它不是餐厅推荐器，也不会读取定位或替你下单。你只需要告诉它这是哪一餐、偏好的口味和消费意愿，它就会从本地菜库中整理候选，通过带动画和音效的转盘给出一个明确答案。

[🎡 在线体验](https://jiaxuan-tao.github.io/vibe-coding-lab/what-to-eat/) · [🧭 使用方法](#-使用方法) · [🔒 本地隐私](#-本地隐私)

## 🎯 它解决什么问题

真正让人纠结的往往不是“去哪家餐厅”，而是还没想清楚今天想吃哪一类、哪一道菜。

这个工具把选择拆成三个动作：

1. 用餐次、口味和预算缩小范围。
2. 选择直接抽一道菜，或先确定菜系再继续细选。
3. 接受结果、再转一次，或把不想吃的菜暂时移出候选。

默认菜库包含中餐、西餐、日料、韩餐、东南亚、快餐小吃、轻食甜品等分类，也覆盖川湘菜、粤菜、江浙菜、火锅烧烤、面食等中餐细分类。

## 🧭 使用方法

### 直接开转

适合“什么都可以，只想快点得到答案”的时候。

1. 选择餐次、口味和消费意愿，也可以保持不限。
2. 查看右侧根据条件生成的候选菜单。
3. 点击转盘中央的“开转”。
4. 在结果小票中选择“就吃这个”“再转一次”或“暂时不要”。

候选池充足时，每轮会随机抽取 8 至 12 个候选；最近接受的 3 个结果不会立即重复出现。

### 按菜系选

适合“想吃中餐，但还不知道具体吃什么”这类模糊需求。

1. 第一轮从中餐、西餐、日料、韩餐等大类中选一个方向。
2. 第二轮继续选择细分菜系。
3. 确定方向后，再从对应菜品中转出最终结果。

每一步都会显示当前选择路径，可以返回上一级重新决定。

## ✨ 核心功能

- 🎡 Canvas 绘制的动态转盘，结果与指针落点保持一致。
- 🥢 “直接开转”和“按菜系选”两种决策模式。
- 🧾 餐次、口味、消费意愿三组快速筛选。
- 🍱 内置多级菜品分类和丰富的预设菜品。
- ⭐ 收藏常吃菜品，并添加自己的菜名和标签。
- 🙅 将暂时不想吃的结果隐藏 24 小时，也可以提前恢复。
- 🔁 避免最近 3 次已接受结果连续重复。
- 🔊 使用浏览器 Web Audio API 合成短促转动与揭晓音效，不加载外部音频文件。
- 📱 支持桌面与移动端布局、键盘操作和减少动态效果偏好。

## 🗂️ 我的菜库

点击页面右上角的菜库按钮，可以管理四类内容：

| 分类 | 用途 |
| --- | --- |
| **预设** | 浏览内置菜品，并将喜欢的菜加入收藏 |
| **收藏** | 集中查看个人常吃菜品 |
| **自定义** | 添加自己的菜名、大类、餐次、口味和消费意愿 |
| **暂时不要** | 查看当前被隐藏的菜品，并手动恢复 |

## 🔒 本地隐私

项目没有账号、后端、数据库和用户分析服务。

筛选条件、自定义菜品、收藏、临时屏蔽记录、最近选择和音效开关只保存在当前浏览器的 `localStorage` 中，不会上传到服务器。清理浏览器站点数据后，这些个性化内容也会一并清除。

## 🛠️ 本地运行

项目没有运行时依赖，只需要支持 ES Modules、Canvas 和 Web Audio API 的现代浏览器。

```bash
git clone https://github.com/jiaxuan-tao/vibe-coding-lab.git
cd vibe-coding-lab/what-to-eat
python3 -m http.server 5176 --bind 127.0.0.1
```

然后打开 [http://127.0.0.1:5176/](http://127.0.0.1:5176/)。

运行自动测试：

```bash
npm test
```

## 🧱 技术实现

- **页面与交互**：HTML、CSS、Vanilla JavaScript
- **转盘绘制**：Canvas 2D API
- **随机选择**：候选池洗牌与浏览器安全随机数
- **音效**：Web Audio API 实时合成，不依赖音频资源
- **本地数据**：带版本校验和异常恢复的 `localStorage`
- **自动测试**：Node.js 原生测试运行器
- **部署方式**：GitHub Pages

项目采用纯静态结构，不需要构建步骤，也不依赖第三方 UI 框架。

## ⚠️ 能力边界

- 结果来自预设菜库和用户自定义内容，不代表附近一定有对应餐厅。
- 工具不会读取定位、营业时间、配送范围、价格或外卖平台库存。
- 口味与预算标签是简化分类，同一道菜在不同城市和商家可能差异很大。
- 随机结果用于减少选择成本，不是营养、过敏原或健康建议。
- 浏览器安全随机数改善了抽取公平性，但不会让主观标签变成客观推荐。
