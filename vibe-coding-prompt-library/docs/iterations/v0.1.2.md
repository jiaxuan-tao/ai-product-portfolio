# v0.1.2｜本地数据迁移

## 目标

为本地优先的提示词资料库增加数据备份与恢复能力，降低更换浏览器、清理站点数据或迁移设备时丢失自定义提示词的风险。

## 本次范围

- 导出当前全部提示词为 JSON 文件。
- 导出文件包含 `schemaVersion: 1`、导出时间和提示词列表。
- 导入符合 v1 格式的 JSON 文件。
- 导入采用合并策略，保留当前资料库中的提示词。
- 遇到相同 ID 时跳过导入，不覆盖已有内容。
- 跳过缺少 ID、标题、分类或正文的无效记录。
- 明确导入与导出只处理浏览器本地数据，不上传网络。

## 数据格式

```json
{
  "schemaVersion": 1,
  "exportedAt": "2026-07-10T00:00:00.000Z",
  "prompts": []
}
```

每个提示词沿用现有字段：`id`、`title`、`category`、`description`、`body`、`isDefault`、`createdAt`、`updatedAt`。本次不改变提示词数据结构。

## 验证重点

- 新导出的 JSON 可以被当前版本重新导入。
- 导入新记录后，刷新页面仍能保留。
- 重复 ID 不会覆盖原有记录。
- 无效 JSON、未知版本或不完整记录会被拒绝或跳过，并展示中文反馈。
- 恢复默认模板、搜索、保存和复制等既有功能保持可用。

## 未包含内容

- 云同步、登录、后端和 AI API。
- 标签、收藏、最近使用和自定义分类管理。
- 与第三方提示词工具的数据互通。

## 后续迭代

计划在 v0.2.0 增加标签、收藏和自定义分类，并让新的导入格式继续兼容 `schemaVersion: 1` 备份文件。
