# 今天吃什么：中国市场菜系重构实施计划

**目标：** 在不读取位置的前提下，完成“中餐 / 外国菜 → 菜系 → 具体菜品”的完整流程，并修复用户确认的结果图、弹窗、指针和底栏问题。

**技术栈：** Vanilla JavaScript、ES Modules、HTML、CSS、Node.js test、Playwright、本地图片资源。

## 任务 1：回归测试

- 更新 `foods`、`picker`、`storage` 和 UI 契约测试。
- 覆盖默认菜系、可选菜系、三段路径、状态迁移、无定位、无底栏、无出餐时间和本地图片。
- 先运行测试，确认因新功能尚未实现而失败。

## 任务 2：数据与选择逻辑

- 重构内置菜品的 `origin`、`cuisine`、`category` 和 `availability`。
- 增加印度菜、墨西哥菜和中东菜内置菜品包。
- 修改直接模式和菜系模式候选池，默认排除未启用的小众菜系。
- 将菜系模式改为方向选择后直接转菜系，再转具体菜品。

## 任务 3：界面与持久化

- 增加“中餐 / 外国菜”方向选择控件。
- 在菜库增加菜系管理开关并持久化。
- 删除底部状态条和结果时间。
- 重写中间结果按钮与状态回退行为。
- 加长加粗高对比筷子指针。

## 任务 4：菜品图片

- 建立 `assets/dishes` 与 `assets/cuisines` 本地资源目录。
- 以现有复古食堂海报为视觉参考，制作统一画幅的细节菜品图。
- 具体菜品绑定独立图片，菜系绑定代表拼盘图。
- 增加图片预加载、错误回退与可访问文本。

## 任务 5：Loop Engineering 验证

- 运行全部 Node 测试、语法检查和 `git diff --check`。
- 启动本地服务，使用 Playwright 在多种桌面与移动端尺寸检查一屏布局、正圆、按钮重叠和弹窗。
- 自动走通直接模式、中文菜系、默认外国菜、启用小众菜系、具体菜品决策、音效与菜库。
- 修复发现的问题，重复验证直到通过。
- 更新 README 和预览图，提交、推送并在线验证 GitHub Pages。
