# 今天吃什么 Design Specification

## 1. Product Purpose

“今天吃什么”是一个纯前端的趣味食物决策工具。它帮助已经决定“要吃饭”，但不想继续浏览菜单或反复比较的人，通过一次直接旋转或一段分层选择快速得到一个具体结果。

产品核心不是餐厅推荐、菜谱搜索或营养建议，而是把低风险的日常选择变成一个有仪式感、可以接受结果的过程。

## 2. Product Boundaries

### Included

- 一个占据主视觉的大转盘。
- `直接开转` 与 `按菜系选` 两种模式。
- 餐次、口味状态和消费意愿三组轻量条件。
- 预置中文菜品库，覆盖中餐、西餐、日料、韩餐、东南亚、快餐小吃、轻食甜品。
- 中餐等大类支持“类别 → 子菜系 → 具体菜品”的分层选择。
- 收藏、自定义菜品、临时屏蔽、最近结果和常用组合。
- 最近三次结果不重复。
- 转盘刻度音与结果提示音，全部由 Web Audio API 在本地合成。
- 本地持久化、响应式布局、键盘操作和降低动态效果支持。

### Excluded

- 账号、登录、云同步和多人房间。
- 后端、数据库和第三方 API。
- 餐厅、地图、定位、营业状态和外卖平台链接。
- 菜谱、营养分析、热量计算和医疗饮食建议。
- AI 推荐、菜单扫描和图片识别。
- 真实价格承诺；消费意愿只使用“简单吃、正常吃、吃顿好的”。

## 3. Primary User Flows

### Flow A: Direct Spin

1. 用户打开页面，默认进入 `直接开转`。
2. 系统根据当前时间预选餐次，其他条件默认为“不限”。
3. 用户可以直接点击转盘中央的“开转”。
4. 转盘从符合条件的菜品中选取 8–12 个候选，并完成减速旋转。
5. 菜单票据从转盘下方滑出，显示具体菜名、所属菜系和标签。
6. 用户选择“就吃这个”“再转一次”或“暂时不要”。

### Flow B: Hierarchical Cuisine Spin

1. 用户切换到 `按菜系选`。
2. 第一轮转盘显示中餐、西餐、日料、韩餐、东南亚、快餐小吃、轻食甜品等大类。
3. 选中大类后，菜单票据提供“就吃这个类别”与“继续选”。
4. 继续后进入子菜系；例如中餐进入川湘菜、粤菜、江浙菜、北方菜、西北菜、云贵菜、广西风味、火锅烧烤、面食。
5. 再次继续后进入符合该子菜系的具体菜品。
6. 顶部选择路径持续显示当前位置，并允许回到上一层。

### Flow C: Personal Library

1. 用户打开“我的菜库”抽屉。
2. 在“预设、收藏、自定义、暂时不要”四个视图中查看和管理选项。
3. 用户可以添加自定义菜名并选择大类、餐次、口味和消费意愿。
4. 所有修改写入当前浏览器的 `localStorage`。

## 4. Candidate Generation

- 每次转盘显示 8–12 个候选；候选不足时显示全部，最少需要两个。
- 条件按餐次、口味状态和消费意愿求交集。
- 如果交集少于两个，界面明确提示用户放宽条件，不静默回退到无关食物。
- 最近确认过的三个具体菜品不进入下一次候选池。
- “暂时不要”在当前浏览器中屏蔽 24 小时。
- “收藏”不改变转盘公平性，只用于快速建立个人组合。
- “换一批候选”在当前过滤结果内重新抽取候选，不直接产生最终结果。
- 最终选择使用 `crypto.getRandomValues()`；视觉旋转角度与最终结果一致。

## 5. Preset Food Taxonomy

预置约 100 个具体菜品，数据保持可读、可编辑，不依赖远程接口。

### Top-Level Groups

- 中餐
- 西餐
- 日料
- 韩餐
- 东南亚
- 快餐小吃
- 轻食甜品

### Chinese Sub-Cuisines

- 川湘菜
- 粤菜
- 江浙菜
- 北方菜
- 西北菜
- 云贵菜
- 广西风味
- 火锅烧烤
- 面食

具体菜品包括但不限于螺蛳粉、寿司、火锅、烤肉、拉面、麻辣烫、水煮鱼、回锅肉、烧鹅、煲仔饭、锅包肉、肉夹馍、过桥米线、披萨、意面、汉堡、炸鸡、泰式咖喱和轻食沙拉。

## 6. Interface Architecture

### Main Stage

- 顶部深绿色横栏包含产品名、模式切换、音效按钮和菜库按钮。
- 横栏下方是三组快速条件，以菜单标签形式展示。
- 中央是陶瓷与套色印刷质感的大转盘。
- 左侧在桌面端显示编号与选择路径。
- 右侧在桌面端显示“今日候选菜单”和“换一批候选”。
- 底部状态栏显示当前候选数量与最近不重复状态。
- 移动端将选择路径折叠到转盘上方，候选菜单和菜库使用底部抽屉。

### Result Ticket

- 不使用与上下文割裂的标准模态框。
- 转盘停止后降低饱和度，菜单票据从转盘下方滑出。
- 菜系中间结果显示“就吃这个菜系”和“继续选一道菜”。
- 具体菜品结果显示“暂时不要”“再转一次”和“就吃这个”。
- 具体菜品结果包含本地视觉素材、菜名、所属菜系及条件标签。

### Food Library Drawer

- 桌面端从右侧滑出，移动端从底部滑出。
- 使用四个标签页：预设、收藏、自定义、暂时不要。
- 自定义表单仅包含菜名、大类、餐次、口味和消费意愿。
- 空状态、重复菜名、字段缺失和候选不足均提供可执行反馈。

## 7. Visual Direction

采用“趣味食堂”方向：

- 背景为 18px 细密墨绿印刷网格，线条细但具有可见对比。
- 主色为墨绿、朱红、芥末黄、米白，并用少量蓝色区分转盘扇区。
- 视觉语言参考复古食堂海报、套色印刷、菜单票据和游戏棋盘。
- 阴影使用明确的错位硬阴影，不使用发光、渐变球或玻璃卡片。
- 转盘具备陶瓷边缘、内圈刻度和筷子式指针。
- 页面不使用营销 Hero、功能介绍卡片或首屏教程。
- 结果视觉素材采用本地位图，不依赖运行时第三方图片。

## 8. Motion and Sound

- 转盘旋转 3.2–4.2 秒，使用逐渐减速的 easing。
- 指针经过扇区边界时触发短促合成刻度音。
- 结果使用两到三个合成音符，不加载 MP3。
- 音效只能在用户点击后创建或恢复 `AudioContext`。
- 音效设置持久化；界面使用标准音量图标并提供可访问名称和 tooltip。
- `prefers-reduced-motion: reduce` 时缩短旋转与票据动画，停用装饰性抖动。

## 9. Persistence

`localStorage` 保存：

- 用户的模式与快速条件。
- 收藏菜品。
- 自定义菜品。
- 临时屏蔽项及过期时间。
- 最近确认结果。
- 音效开关。
- 常用组合。

读取失败或数据结构损坏时回退到安全默认值，不阻塞转盘。

## 10. Technical Architecture

- HTML：页面语义、对话框/抽屉和表单结构。
- CSS：趣味食堂视觉、响应式布局、转盘状态和动画。
- `foods.js`：预置菜品及筛选标签。
- `picker.js`：过滤、候选抽样、分层路径、随机选择和历史排除。
- `storage.js`：版本化本地数据读写和校验。
- `audio.js`：Web Audio API 合成刻度音与结果音。
- `app.js`：DOM 状态、事件绑定、转盘渲染、抽屉与结果流程。
- Node.js 原生测试：覆盖核心纯函数、持久化校验和页面契约。

项目不增加运行时依赖，不需要构建步骤。

## 11. Open-Source Adaptation

转盘 Canvas/旋转数学参考 `Kayleeli/food-spinner` 的 MIT 开源实现，但产品信息架构、菜品数据、分层流程、视觉、持久化、音效和结果交互重新实现。

项目内保留原项目 MIT 许可证副本和来源说明；仓库 README 明确说明复用了什么、修改了什么。

## 12. Acceptance Criteria

- 用户无需阅读说明即可在首屏完成一次旋转。
- 两种模式均可从开始走到可接受的具体结果。
- 中餐路径至少支持大类、子菜系和具体菜品三层。
- 过滤条件、最近三次排除和临时屏蔽均通过自动测试。
- 自定义菜品刷新页面后仍存在。
- 音效关闭后不再产生音频，刷新后设置保持。
- 桌面宽度 1440px、平板宽度 768px、手机宽度 390px 下无重叠和横向滚动。
- 键盘可以操作模式、筛选、开转、结果操作和菜库。
- 降低动态效果设置生效。
- 本地测试通过，GitHub Pages 工作流复制完整项目资源。
- 总仓库 README、项目目录和展示页包含新项目及在线链接。
