# 今天吃什么：中国市场菜系与选择路径设计

## 目标

把当前混合“菜系、国家和用餐形式”的转盘，改成符合中国用户认知且能真正走通的三段选择：

1. 选择饮食方向：中餐或外国菜。
2. 在该方向下选择菜系。
3. 在该菜系下选择具体菜品。

应用不读取位置，不请求定位权限，也不根据城市自动判断附近餐厅。

## 核心选择路径

### 直接开转

直接从全部已启用菜品中抽取候选。餐次、口味和预算筛选继续生效。

默认包含：

- 所有常见中餐菜系。
- 日料、韩餐、西餐。

仅当用户在菜库中主动启用后，才包含泰国菜、越南菜、东南亚其他、印度菜、墨西哥菜和中东菜。

### 按菜系选

1. 页面先提供“中餐 / 外国菜”两个明确的方向按钮。
2. 转盘显示该方向下可用的菜系。
3. 转到菜系后，结果弹窗只提供“重新选菜系”和“继续选一道菜”。
4. 继续后，转盘只显示该菜系的具体菜品。
5. 只有具体菜品结果才出现“暂时不要 / 再转一次 / 就吃这个”。

中间步骤不再提供“就吃这个菜系”，避免用户以为已经完成选餐。

## 菜系结构

### 中餐默认菜系

- 川湘菜
- 粤菜
- 江浙菜
- 东北菜
- 西北菜
- 云贵菜
- 广西风味
- 火锅烧烤
- 粉面米线
- 中式小吃
- 中式甜品

### 外国菜默认菜系

- 日料
- 韩餐
- 西餐

### 外国菜可选菜系

- 泰国菜
- 越南菜
- 东南亚其他
- 印度菜
- 墨西哥菜
- 中东菜

可选菜系使用内置菜品包。用户在“菜库 > 菜系管理”中一键启用或关闭，无需手工逐道录入。已有“自定义菜品”能力保留，作为补充。

## 数据模型

每道菜包含：

- `origin`：`中餐` 或 `外国菜`。
- `cuisine`：第二层菜系。
- `category`：更细的展示分类，可选。
- `availability`：`default` 或 `optional`。
- `image`：本地菜品图片路径。
- 餐次、口味、预算标签。

旧的 `group` 字段在读取自定义菜品时兼容，但内置数据不再依赖它决定层级。

## 结果视觉

- 删除底部状态条。
- 删除结果弹窗中的“出餐时间”。
- 具体菜品结果使用对应本地图片，统一为俯拍、复古食堂海报质感、相同画幅和色调。
- 菜系结果使用该菜系的代表菜拼盘图。
- 自定义菜品没有图片时，使用统一的本地食物海报占位。
- 图片加载失败时也回退到本地占位图，不出现空白。

## 转盘指针

筷子指针加长加粗，使用米白主体、墨绿描边和朱红尾部，与黄、红、蓝、绿转盘扇区均保持高对比。指针尖端继续对准十二点方向，结果索引计算不变。

## 持久化

本地状态新增 `enabledOptionalCuisines`，只保存用户启用的小众菜系。默认值为空数组。旧版本状态自动迁移，收藏、暂时不要、自定义菜品和音效设置继续保留。

## 验收标准

- 不请求定位权限。
- 菜系模式可以完整走通“方向 → 菜系 → 菜品”。
- 任何中间结果都不会提前结束流程。
- 小众外国菜默认不出现在转盘中，启用后立即可选，刷新后仍保留。
- 桌面常见尺寸下主页面一屏展示且无主体滚动条。
- 结果弹窗无“出餐时间”，每道内置菜有对应本地图片。
- 指针清晰可见，转盘为正圆，最终结果与指针位置一致。
